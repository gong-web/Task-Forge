# GIF 录制操作步骤

本文档详细说明每段功能演示 GIF 的录制流程，包括具体点击位置、输入内容、操作顺序，以及录制软件的参数设置。

---

## 一、录制软件设置

推荐使用 **ScreenToGif**（免费开源，[https://www.screentogif.com/](https://www.screentogif.com/)）。

### 1.1 启动与参数配置

1. 打开 ScreenToGif，点击主界面 **「录制器」**（Recorder）按钮。
2. 将录制框拖到应用程序窗口上方，调整大小使其覆盖整个 Task Forge 窗口（建议 1200×800 或更宽）。
3. 点击顶部菜单 **「文件」→「选项」**（Options）：
   - **帧率**：设置为 **12 fps**（Frames per second）
   - **捕获模式**：选"屏幕"（Screen capture）
4. 点击 **「录制」**（Record）按钮开始，操作完毕后点击 **「停止」**（Stop / F8）。

### 1.2 导出参数

1. 停止录制后，进入 **「编辑器」**（Editor）界面。
2. 删除首帧和尾帧的空白帧（点击帧后按 Delete）。
3. 点击 **「文件」→「另存为」**（Save as）：
   - 格式选 **GIF**
   - **宽度**：640 px（勾选"保持比例"，高度自动按比例缩放）
   - 编码器：选 **FFmpeg**（画质更好）或系统默认均可
   - 帧率重编：与录制时一致（12 fps）
4. 保存路径：`docs/demo/XX-name.gif`（XX 为两位数序号）

### 1.3 演示数据重置

录制前确认演示数据完整：

```powershell
# 在 Task-Forge 项目根目录执行（关闭应用后操作）
Remove-Item data/task_forge.db -Force
python src/main.py
# 应用启动后 data_seeder 自动植入演示数据，Ctrl+C 关闭
```

重新打开应用即可开始录制。

## 二、录制前检查清单

每次录制前确认：

- [ ] 演示数据已通过 `data_seeder.py` 植入（任务树有内容）
- [ ] 应用窗口最大化（`showMaximized()`）
- [ ] 屏幕分辨率为 1920×1080 或 2560×1440
- [ ] ScreenToGif 帧率已设置为 12 fps
- [ ] 录制框已覆盖整个应用窗口
- [ ] 关闭其他通知弹窗和覆盖物

## 三、逐段录制操作步骤

### GIF 01 — 今日视图与任务树（docs/demo/01-today-view.gif）

**目标效果**：展示默认启动视图，包含今日任务树、逾期高亮、子任务展开。

**操作步骤**：

1. 启动 ScreenToGif 录制。
2. 应用启动后默认显示 **今日视图**（左侧导航栏「今日」选项高亮）。
3. 鼠标缓慢从上到下浏览任务树，停留约 1.5 秒。
4. 找到一个有子任务的父任务节点（示例演示数据中有多个），单击左侧 **三角展开箭头** 展开子任务列表。
5. 再展开另一个包含 3 个以上子任务的父任务。
6. 将鼠标悬停在标记为红色的逾期任务上（红色文字/背景任务），停留约 1 秒。
7. 将鼠标悬停在标记为橙色的今日到期任务上，停留约 1 秒。
8. 缓慢向下滚动任务列表展示内容丰富性，然后滚回顶部。
9. 停止录制。

**时长参考**：8–12 秒。

### GIF 02 — 任务编辑器与上下文注入（docs/demo/02-create-task.gif）

**目标效果**：展示新建任务流程，时间约束实时联动，标签输入，父子关系设置。

**操作步骤**：

1. 启动 ScreenToGif 录制。
2. 点击左侧导航栏 **「新建任务」**（或主工具栏上的 **「+」** 按钮，具体取决于 UI）进入编辑器。
3. 在 **「标题」** 输入框（空白文本框，顶部最显眼的输入框）中单击，输入：`完成项目报告初稿`
4. 点击 **「优先级」** 下拉框，选择 **「高」**。
5. 勾选 **「截止时间」** 复选框（或点击截止时间启用开关），日期时间选择器激活。
6. 在日期输入框中将日期修改为 **明天的日期**（点击"+"上调一天）。
7. 在小时滚轮上向下滚动，将截止时间设为 **17:00**。
8. 勾选 **「提醒」** 复选框，观察提醒时间自动填充为截止时间前的值。
9. 将提醒时间的小时数上调到 **16:00**（比截止时间早 1 小时）。
10. 在 **「标签」** 输入框中单击，输入 `重要` 后按 **空格键** 确认，再输入 `学习` 后按空格键，观察标签气泡出现。
11. 在 **「分类」** 下拉框中选择 **「学习」**。
12. 点击 **「完成」** 或 **「保存」** 按钮提交任务。
13. 观察任务出现在今日视图任务树中（约 0.5 秒动效）。
14. 停止录制。

**时长参考**：15–20 秒。

### GIF 03 — 父子 DAG 联动与自动归档（docs/demo/03-parent-child-dag.gif）

**目标效果**：展示多个子任务完成后父任务自动归档的联动效果。

**操作步骤**：

1. 在今日视图中，找到一个有 **3 个子任务** 的父任务，展开子任务列表。
2. 启动 ScreenToGif 录制。
3. 单击第一个子任务左侧的 **复选框**，勾选为完成（观察勾选动效）。
4. 等待约 0.5 秒。
5. 单击第二个子任务左侧的 **复选框**，勾选为完成。
6. 等待约 0.5 秒。
7. 单击第三个（最后一个）子任务左侧的 **复选框**，勾选为完成。
8. 观察：父任务 **自动变为已完成状态**（复选框自动勾选，可能有庆祝动效或绿色高亮）。
9. 停顿约 1.5 秒展示最终状态。
10. 停止录制。

**时长参考**：6–10 秒。

### GIF 04 — 甘特图时间轴浏览（docs/demo/04-gantt-view.gif）

**目标效果**：展示甘特图视图，父子缩进，进度条，时间轴拖动。

**操作步骤**：

1. 启动 ScreenToGif 录制。
2. 点击左侧导航栏 **「甘特图」**（Gantt）选项，切换到甘特图视图。
3. 等待约 0.5 秒让视图完全渲染。
4. 观察甘特图：父任务标题在左，子任务向右缩进约 18px，进度条渲染于行内。
5. 将鼠标缓慢滑过几条甘特条（不点击），观察其颜色和进度条长度。
6. 向下滚动甘特图列表，展示更多任务行（滚动 2–3 次鼠标滚轮）。
7. 如果甘特图支持水平时间轴拖动，向左拖动时间轴显示更早的任务。
8. 滚回顶部，停顿约 1 秒。
9. 停止录制。

**时长参考**：8–12 秒。

### GIF 08 — 提醒弹窗与延期（docs/demo/08-reminder-dialog.gif）

**目标效果**：展示提醒弹窗出现、声音播放、延期操作。

> **特别说明**：此 GIF 需要预先设置一个 `remind_at` 为即将到来的时间（1–2 分钟后）的任务。

**操作步骤**：

1. 新建一个任务，设置 `remind_at` 为当前时间之后约 **1 分钟**。
2. 保存任务，进入今日视图等待。
3. 在提醒时间到达前 5 秒，启动 ScreenToGif 录制。
4. 等待提醒弹窗自动弹出（应用程序内部对话框）。
5. 弹窗出现后停顿约 1.5 秒（展示弹窗内容：任务标题、时间、操作按钮）。
6. 点击 **「延期 10 分钟」**（或对应的延期按钮），观察弹窗关闭。
7. 停顿约 0.5 秒。
8. 停止录制。

**时长参考**：6–10 秒（不含等待时间，录制从弹窗出现前 5 秒算起）。

## 四、录制后处理

每段录制完成后在 ScreenToGif 编辑器中：

1. 删除首尾多余的空白帧（点击帧，按 Delete）
2. 检查是否有手误操作帧（可逐帧预览删除）
3. 如有超出录制框的光标跳出，可裁剪画面（菜单「图片」→「裁切」）
4. 按照 [1.2 导出参数](#12-导出参数) 导出并保存到 `docs/demo/` 目录
5. 在 README.md 对应的 GIF 占位符上验证引用是否正确
