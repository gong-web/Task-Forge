# 演示脚本

## 一、演示目标

在 5 分钟内完整展示课程作业要求中的全部核心功能，并体现项目已经具备较好的完成度和可扩展性。

## 二、演示前准备

### 1. 演示环境

- Windows 本地桌面环境
- 已安装 Python 与项目依赖
- 能正常显示系统托盘通知

### 2. 演示数据

- 使用 `data/demo_data.json` 作为预置演示数据
- 启动前确认数据库已导入演示数据

### 3. 建议准备的截图或录屏素材

- 启动后主界面截图
- 任务树与子任务结构截图
- 父任务自动完成联动截图
- 提醒弹窗截图
- 搜索筛选截图
- 便签编辑截图
- 导出与导入恢复截图

## 三、5 分钟演示流程

### 第 1 步：启动并展示自动加载

时长建议：30 秒

- 启动程序：`python src/main.py`
- 展示状态栏提示“已自动加载 N 条任务、N 条便签”
- 说明项目会自动加载历史数据，且数据保存在本地 SQLite 中

### 第 2 步：展示任务地图与子任务结构

时长建议：60 秒

- 展示中部“任务地图”树结构
- 点击一个父任务，展示右侧“专注面板”
- 说明支持多层级父子任务
- 展示底部层级路径

### 第 3 步：展示父任务自动联动完成

时长建议：30 秒

- 选择一个还有未完成子任务的父任务
- 将最后一个未完成子任务标记为完成
- 展示父任务自动变为完成状态
- 说明这是课程要求中的关键逻辑

### 第 4 步：展示提醒功能

时长建议：60 秒

- 选择一个带提醒的任务
- 展示左侧“即将提醒”列表
- 等待提醒触发或使用已到期提醒数据
- 展示托盘通知
- 点击“提醒延后 10 分钟”，说明可继续处理

### 第 5 步：展示搜索、筛选、编辑、删除

时长建议：40 秒

- 在左侧输入关键词进行搜索
- 切换状态筛选或优先级筛选
- 编辑一个任务的分类或描述
- 删除一个演示任务，说明删除后子任务会一起移除

### 第 6 步：展示便签功能

时长建议：20 秒

- 切换到右侧“便签板”
- 新建或编辑一条便签
- 演示置顶功能

### 第 7 步：展示导出与导入恢复

时长建议：30 秒

- 导出当前数据到 JSON
- 说明支持备份恢复
- 如时间允许，可删除一条任务后再导入恢复

### 第 8 步：展示 AI 使用说明

时长建议：60 秒

- 打开 `docs/AI使用说明.md`
- 简要说明使用的 AI 工具、使用环节、提示词模板、优缺点总结
- 强调 AI 只是辅助工具，最终内容经过人工审核与调整

## 四、推荐讲解词

### 开场

“这是我实现的课程作业项目 Task Forge，它是一个带图形界面的个人任务管理器，支持子任务、数据持久化、提醒通知和便签功能。”

### 讲解核心亮点

“这个项目的重点不是普通待办列表，而是层级任务结构和父任务自动联动完成逻辑，这是本次作业里比较关键的要求。”

### 讲解提醒

“应用关闭窗口后不会直接退出，而是最小化到系统托盘，所以提醒功能仍会继续运行。”

### 讲解 AI

“在开发中我使用了 AI 辅助分析需求、重构界面、补全文档和验证代码，但最终方案和作业规范是我人工审核后确定的。”

## 五、演示注意事项

- 演示前先导入 demo 数据，避免现场临时录入耗时
- 提醒任务最好设置为演示时可立即触发
- 状态栏与托盘提示要提前确认可正常显示
- 若现场网络不稳定，不影响项目，因为本项目为本地桌面应用
- 答辩时优先突出“子任务联动”“实时保存”“提醒通知”“AI 使用说明”四点
